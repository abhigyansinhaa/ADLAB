Introduction

Rust is a systems programming language focused on safety, speed, and concurrency. It accomplishes these goals without having a garbage collector, making it a useful language for a number of use cases other languages aren't good at.

Rust provides powerful guarantees about your program's memory safety without sacrificing performance. Whether you're building command-line tools, web services, or embedded systems, Rust gives you the control and reliability you need.

Why Rust? Memory safety without garbage collection, concurrency without data races, and abstraction without overhead.

Getting Started

To get started with Rust, you'll need to install the Rust toolchain. The recommended way is through rustup, Rust's official installer and version management tool.

Once installed, you can verify your installation by running:

rustc --version
cargo --version

Cargo is Rust's build system and package manager. Most Rustaceans use this tool to manage their Rust projects because Cargo handles a lot of tasks for you, such as building your code and downloading dependencies.

Variables and Mutability

By default, variables in Rust are immutable. This is one of Rust's nudges to write code that takes advantage of safety and easy concurrency.

You declare variables using the let keyword:

let x = 5;        // immutable
let mut y = 10;   // mutable

y = 15;           // OK: y is mutable
// x = 6;         // Error: x is immutable

The distinction between mutable and immutable variables helps you think about when data changes and when it doesn't, leading to fewer bugs.

  • Use let for immutable bindings
  • Add mut keyword for mutable variables
  • Constants use const and must have type annotations
Data Types

Every value in Rust has a data type, which tells Rust what kind of data is being specified. Rust is a statically typed language, meaning it must know the types of all variables at compile time.

Rust has two data type subsets: scalar and compound types.

Scalar Types:

  • Integers: i8, i16, i32, i64, i128, u8, u16, u32, u64, u128
  • Floating-point: f32, f64
  • Boolean: bool (true or false)
  • Character: char (Unicode scalar value)

Functions

Functions are pervasive in Rust code. The fn keyword allows you to declare new functions. Rust uses snake_case as the conventional style for function names.

fn main() {
    println!("Hello, world!");
    another_function(5);
}

fn another_function(x: i32) {
    println!("The value of x is: {}", x);
}

Function parameters must have their types declared. This is a deliberate decision in Rust's design that helps the compiler give more helpful error messages.

Functions can return values using the -> syntax to declare the return type:

fn add(a: i32, b: i32) -> i32 {
    a + b  // No semicolon = return value
}
Control Flow

The ability to run code depending on conditions is fundamental to most programming languages. Rust provides if expressions and loops for control flow.

let number = 6;

if number % 2 == 0 {
    println!("Even");
} else {
    println!("Odd");
}

// Loop with break
let mut counter = 0;
let result = loop {
    counter += 1;
    if counter == 10 {
        break counter * 2;
    }
};

Rust has three kinds of loops: loop, while, and for. The for loop is particularly powerful and safe for iterating over collections.

Ownership

Ownership is Rust's most unique feature and enables Rust to make memory safety guarantees without needing a garbage collector. Understanding ownership is crucial to becoming proficient in Rust.

The ownership rules are:

  • Each value in Rust has a variable called its owner
  • There can only be one owner at a time
  • When the owner goes out of scope, the value is dropped
fn main() {
    let s1 = String::from("hello");
    let s2 = s1;  // s1 is now invalid (moved)
    
    // println!("{}", s1);  // Error!
    println!("{}", s2);     // OK
}

This system prevents data races at compile time and eliminates many common bugs found in other systems programming languages.
